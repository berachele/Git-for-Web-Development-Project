    37. Walk me through Lambda's git flow. 
        Fork your assignment, (should see it succeed when it shows your name and 'forked from:'), add TL as collaborator
        copy clone link in github, in your Terminal cd to where you want your clone to be, git clone [clone link], 
        cd into newly created repo, create new branch under your name (git checkout -b [first-lastName]), 
        Work on your repo!, when you make changes: git add . to add into committing link, then git commit -m "with a 
        specific message about what changes were made", git push -u origin [your branch-name], Submit a PR! (Pull Request),
        Go to your forked project in github--click branch:master button; click your branch-name; click New pull request
        (will redirect you to PR page), add your TL as a reviewer before creating!!, change base repo to your forked repo,
        Click create repo--do not merge though!
